Raw string :

A raw string, also known as a raw literal or raw text literal, is a way to represent strings in programming languages, where escape sequences are not processed or interpreted. In many programming languages, escape sequences are used to represent special characters, such as newline characters (\n) or tabs (\t), within a string. However, in a raw string, these escape sequences are treated as literal characters, and their backslashes are not used for escaping.

The exact syntax for creating raw strings may vary from one programming language to another, but in many languages, an “r” (or some other character) is placed in front of the string literal to indicate that it should be treated as a raw string. Indexing :

In Python, indexing is the process of accessing individual elements of a data structure, such as a list, tuple, string, or other sequence-like objects. Indexing allows you to retrieve specific items from the collection by specifying their position within the sequence. Python uses zero-based indexing, which means the first element is at index 0, the second element is at index 1, and so on.

Here’s how indexing works for various data structures:

  • Indexing in Strings:

Strings are sequences of characters, and you can access individual characters by their index.

  • Negative Indexing:

You can also use negative indices to access elements from the end of a sequence. -1 refers to the last element, -2 refers to the second-to-last element, and so on.

  • Slicing
  • Slicing is a more versatile way to extract a portion of a sequence. You specify a range of indices to create a new sequence containing elements from the original.
  • To slicing we need 3 values [Starting index : ending index : step size]

Keep in mind that attempting to access an index that doesn’t exist will result in an “Index Error.” It’s important to ensure that the index you use is within the valid range for the data structure.
